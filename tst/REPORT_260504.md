## 5. Technical Patterns and Lessons Learned

### Isabelle-Specific Patterns
- **`define` vs `let`**: Use `define` (not `let`) for terms used in `OF` chains. `let` creates opaque abbreviations that prevent unification.
- **Deep OF chains**: Use `proof (rule Lemma[OF a b])` with explicit `show` blocks instead of `by (rule Lemma[OF a b c d e])` which times out.
- **by100 timeout discipline**: Extract quantified facts with `[rule_format, OF]` before blast; decompose large goals into small steps.
- **Function extensionality**: `f ∘ g = (λx. f(g x))` via `(rule ext)` + `simp`; for pairs use `(rule ext, cases x)` + `simp`.
- **Affine continuity**: Use `top1_continuous_map_on_real_subspace_open_sets` + `continuous_intros`.
- **Topology open sets**: Use `is_topology_on_def` finite intersection axiom with named facts; avoid unfolding directly in blast.

### Proof Architecture
- **Lifting criterion**: ~500 lines, the largest single proof. Well-definedness uses loop extraction + Theorem_54_3 + reparametrized lifts + Lemma_54_1_uniqueness. Continuity uses local path-connectivity + covering sheets + homeomorphism inverses.
- **Covering space classification**: Chain: lifting criterion → 79.2 → 79.4 → 80.3. Each link is proved except for specific algebraic/topological sub-steps.
- **SvK**: The parameterized version is the correct formalization. The existential version has a fundamental type-variable issue and was abandoned with `oops`.
